## 1. 컬럼 복구 결정

### 결정 #1: `mer_stop` 복구

**원래 제거 근거**
"휴업 가맹점 — 분석 기준 불명확, mer_total과 이중집계 우려"

**재검토 계기**
"잠재적 폐업 위험군 분석 시 보조 지표로 활용 가치 있음"이라는 판단 → 보수적 원칙에 따라 데이터로 직접 검증

**검증 방법 및 결과**

```python
# 원본 집계로 직접 비교 (2023년 1월 분당구)
merge_card_merchant[(ym==202301) & (gu_nm=='분당구')]['mer_cnt'].sum()   # 53,305
merge_card_merchant[(ym==202301) & (gu_nm=='분당구')]['stop_cnt'].sum()  # 24,945
master[(ym==202301) & (gu_nm=='분당구')]['mer_total']                    # 53,305 ✅ 일치
master[(ym==202301) & (gu_nm=='분당구')]['mer_stop']                     # 24,945 ✅ 일치
```

- **결론:** `mer_stop`은 `mer_total`에 **포함**되어 있음 (이중집계 확인됨)
- **대응:** 제거 대신 포함 관계를 파악하고 파생 컬럼(`mer_active`)으로 적극 활용
- **추가 발견:** 2024→2025년 사이 3구 모두 휴업률 3~4%p 급등 (분석 포인트 확보)

| 연도 | 분당구 | 수정구 | 중원구 |
|------|--------|--------|--------|
| 2023 | 48.0% | 42.0% | 47.4% |
| 2024 | 48.2% | 42.2% | 46.8% |
| 2025 | **51.3%** | **46.0%** | **48.9%** |

**최종 결정:** ✅ 복구

### 결정 #2: `mer_fran` 복구

**원래 제거 근거**
"프랜차이즈 — 독립 자영업 분석 포커스와 무관"

**재검토 계기**
"프랜차이즈 사장님도 자영업자"라는 관점 재고 + 보수적 원칙 적용

**검증 방법 및 결과**

```python
# 원본 집계로 직접 비교 (2023년 1월 분당구)
merge_card_merchant[(ym==202301) & (gu_nm=='분당구')]['fran_cnt'].sum()  # 4,178
master[(ym==202301) & (gu_nm=='분당구')]['mer_fran']                     # 4,178 ✅ 일치
# mer_fran / mer_total = 4,178 / 53,305 = 7.8%
```

- **결론:** `mer_fran`은 `mer_total`에 **포함**되어 있음 (확인됨)
- **추가 발견:** 프랜차이즈 비율(fran_rate_pct)과 대출연체율(loan_delinq_rate) 상관계수 **r = -0.803** → 프랜차이즈 비중 높은 구일수록 금융 취약성 낮음

| 구 | 프랜차이즈 비율 | 대출연체율(‰) |
|----|----------------|--------------|
| 분당구 | 8.5% | 2.34 |
| 수정구 | 5.9% | 4.19 |
| 중원구 | 4.8% | 4.56 |

> ⚠️ **해석 주의:** 이 상관관계는 "분당구가 부유해서"라는 구 규모 효과일 수 있음. Phase 2에서 구 내부 시계열 분석으로 인과관계 검증 필요.

- **대응:** 제거 대신 `mer_indep = mer_total - mer_fran`으로 독립 자영업 수를 분리하여 분석 초점 유지

**최종 결정:** ✅ 복구

### 결정 #3: `mer_op_lt1y`, `mer_op_1_2y`, `mer_op_2_3y`, `mer_op_3_4y`, `mer_op_4_5y` 복구 (5개)

**원래 제거 근거**
"mer_op_5yp와 고상관 (r=0.95~0.97+), 정보 중복"

**재검토 계기**
고상관이라도 정보가 실질적으로 중복되는지 검증 필요 → 보수적 원칙 적용

**검증 방법 및 결과**

```python
# 전체 데이터 vs 구 내부 시계열 상관 비교
전체(3구 혼합):   mer_op_lt1y vs mer_op_5yp r = 0.958  ← 높음
분당구 내부만:    mer_op_lt1y vs mer_op_5yp r = 0.235  ← 낮음 (독립 정보!)
수정구 내부만:    mer_op_2_3y vs mer_op_5yp r = -0.051 ← 사실상 무관!
```

- **결론:** 전체 데이터의 고상관은 **구 규모 스케일 효과**(분당구가 모든 코호트에서 약 2.3배 큼)에서 비롯된 **허위 상관관계**였음
- 구 내부 시계열에서는 독립 정보 확인됨 (r 최저 -0.051)

**생존 퍼널 분석 가치**

이 5개 컬럼을 연결하면 창업 후 단계별 탈락률을 계산할 수 있음:

| 생존 구간 | 분당구 | 수정구 | 중원구 |
|-----------|--------|--------|--------|
| 1→2년 | 91.3% | **87.1%** | 86.5% |
| 2→3년 | **84.3%** | 86.2% | 84.7% |
| 3→4년 | 88.3% | **84.8%** | 86.1% |
| 4→5년 | 89.2% | 86.6% | **90.3%** |

> ⚠️ **해석 주의:** 이 수치는 같은 가게를 추적한 종단(longitudinal) 데이터가 아니라, 특정 시점의 운영 연령 분포 **단면(cross-section)**임. 따라서 순수한 코호트 생존율로 직접 해석하면 안 되며, 구간 비교 및 시계열 추이 분석에 활용해야 함.

**최종 결정:** ✅ 복구 (5개 전량)

| **검증 방법** | 구별 연도별 추이 계산, 이상 패턴 없음 확인 |

## 3. 데이터 가정 사항

### 가정 A: `mer_total` 집계 기준 (신뢰도: 높음 ✅)

- **가정 내용:** `mer_total` = 영업중 + 휴업 + 프랜차이즈 (중복 집계 없음)
- **근거:** 원본 파일(`merge_card_merchant.csv`)의 `mer_cnt` 집계가 마스터의 `mer_total`과 정확히 일치함 (2023.01 분당구 기준 53,305 = 53,305)
- **잘못될 경우 영향:** mer_active, mer_indep 계산 전체가 왜곡됨 (높음)
- **검증 상태:** ✅ 검증 완료

---

### 가정 B: `mer_stop` 집계 기준 (신뢰도: 높음 ✅)

- **가정 내용:** `mer_stop` = 휴업 상태 가맹점 수이며, `mer_total` 안에 포함됨
- **근거:** 원본 `stop_cnt` 집계가 마스터 `mer_stop`과 정확히 일치 (24,945 = 24,945), mer_total - mer_stop으로 영업중 수 계산 가능
- **잘못될 경우 영향:** `mer_active` 계산이 부정확해짐 (높음)
- **검증 상태:** ✅ 검증 완료

---

### 가정 C: `mer_fran` 집계 기준 (신뢰도: 높음 ✅)

- **가정 내용:** `mer_fran` = 프랜차이즈 가맹점 수이며, `mer_total` 안에 포함됨
- **근거:** 원본 `fran_cnt` 집계가 마스터 `mer_fran`과 정확히 일치 (4,178 = 4,178)
- **잘못될 경우 영향:** `mer_indep` 계산이 부정확해짐 (높음)
- **검증 상태:** ✅ 검증 완료

---

### 가정 D: `mer_op_*` 코호트의 독립 정보성 (신뢰도: 중간 ⚠️)

- **가정 내용:** 운영기간 코호트(lt1y~4_5y)는 mer_op_5yp와 전체 상관이 높더라도, 구 내부 시계열 분석에서는 독립적인 정보를 담고 있음
- **근거:** 구 내부 시계열 상관 분석에서 r 최저 -0.051 확인 (수정구 2_3y vs 5yp)
- **한계:** 스냅샷 데이터이므로 진정한 코호트 생존율로 해석 불가. 같은 가게를 시계열 추적한 게 아님
- **잘못될 경우 영향:** 생존 퍼널 분석 결과가 오해를 줄 수 있음 (중간)
- **검증 상태:** ⚠️ 부분 검증 — Phase 2 통계 검정 시 재확인 권장

---

### 가정 E: `mer_indep` 계산의 근사성 (신뢰도: 중간 ⚠️)

- **가정 내용:** 독립 자영업 가맹점 수 ≈ `mer_total - mer_fran`
- **한계:** mer_stop(휴업) 내에 프랜차이즈 휴업과 독립 자영업 휴업이 혼재. 현재 데이터에서 이를 분리할 방법이 없음. 따라서 "실제 영업 중인 독립 자영업 수"는 정확히 계산 불가
- **근사 공식:** `mer_indep` = 독립 자영업(영업중+휴업), `mer_active` = 전체 영업중(독립+프랜차이즈)
- **잘못될 경우 영향:** 독립 자영업 규모 추정 오차 발생 (낮음~중간 — 프랜차이즈 비율이 5~9% 수준으로 크지 않음)
- **검증 상태:** ⚠️ 한계 인지 후 근사값으로 활용

---

### 가정 F: `tot_people`의 인구 대표성 (신뢰도: 중간 ⚠️)

- **가정 내용:** `tot_people`(신용정보 등록 인구)은 실질적인 지역 경제 인구를 근사적으로 대표함
- **근거:** 원본 `TOT_CNT` 집계와 마스터 `tot_people`이 정확히 일치 확인 (383,300 = 383,300). 분당구 기준 392,848명으로 실제 분당구 인구(약 49만 명)보다 낮지만, 경제 참여층 규모를 반영
- **한계:** 신용카드·대출 이력이 없는 영유아, 무신용 고령자, 외국인 등은 미포함 가능성 있음. 주민등록 인구와 직접 비교 불가
- **잘못될 경우 영향:** non_econ_rate 해석 오류 가능성. 단, 시계열 추이 분석(구 내부 변화)에는 영향 낮음 (중간)
- **검증 상태:** ⚠️ 출처 확인 완료, 대표성 한계는 인지 후 활용

## 3-2. 신규 컬럼 추가 확정 (2026-04-08)

### 신규 #1: `sum_card_use` 추가 확정

**원본 컬럼:** `merge_credit_info.csv` → `SUM_CARD` (소비자 카드 사용액)

**추가 근거**
- `sales_amt`(가맹점 매출 — 공급 측)와 달리 `sum_card_use`는 소비자 실제 카드 지출액(수요 측) → 동일 지역을 다른 시각으로 볼 수 있음
- 지역 소비 활력 지표: sum_card_use 하락 + sales_amt 하락 = 상권 침체 이중 확인 구조
- 자영업자가 많은 카드 사용 지역인지, 카드 소비 대비 가맹점 매출 비율 분석 가능

**집계 방식**
```python
# merge_credit_info → SUM_CARD, ym + gu_nm 기준 SUM 집계
sum_card_use = credit.groupby(['ym','gu_nm'])['SUM_CARD'].sum()
```

**주의사항**
- 단위: 원(₩). 수정구·중원구 vs 분당구 간 절대 규모 차이 큼 → 구별 비교 시 비율 변화(전월 대비 %) 또는 1인당 환산 권장
- 마스터 CSV 확인: 이미 포함되어 있었음 (column-navigator에서 누락 표기된 것을 이번에 정식 확정)
